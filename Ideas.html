Модуль для рисования графики в Си, GraphC.h.

    Цель:

  1. Должен быть получен класс с хорошим интерфейсом для рисования графики в С++, используя ВинАпи, чтобы там были реализованы все графические примитивы, со сглаживанием, без, а так же с прозрачностью и без. Чтобы он был аналогичен модулю GraphABC, и в нем присутствовали события. И чтобы он подходил мне.

    Аналоги:

  1. GraphABC.

    1. Есть плавная графика: lockdrawing, redraw.

    2. Могут возвращаться все переменные, наример цвет пера, цвет кисти, стиль пера и т.д.

    3. Есть все основные графические примитивы: setpixel, line, circle, ellipse, polyline, polygon, rectangle, rounrect, arc, pie, chord, floodfill, TextOut.

    4. Можно настраивать цвет пера, кисти, и есть стандартные цвета, так же есть функция rgb, getred, getgreen, getblue.

    5. Можно настраивать перо: цвет, толщина, стиль.

    6. Можно настраивать кисть: цвет, стиль, фоновое изображение кисти.

    7. Можно настраивать шрифт выводимого текста: имя, размер, цвет, подчеркнутый-жирный и т.д., а так же можно узнать каких размеров будет рисоваться данный текст на экране.

    8. Можно работать с рисунками: создавать, загружать, сохранять, удалять из памяти, рисовать, обрезать, сжимать, загружать рисунок с экрана.

    9. Можно работать с окном: очищать, менять размер, передвигать, изменять заголовок, закрывать, разворачивать и сворачивать.

    10. Можно работать с буффером: перенаправлять всё рисование в буффер, моментально отрисовывать картинку из буффера в окно.

  2. Events.

    1. Есть события на: нажатие, опускание, двигание мыши; нажатие, опускание клавиши; изменение размеров, закрытие окна.

    2. Есть коды вирутальных клавиш на специальные клавиши.

  3. Utils.

    1. Можно работать с каталогами: создавать, переименовывать, удалять, определять существование, определять текущий, менять текущий.

    2. Можно работать с файлами: переименовывать, удалять, определять существование. Так же можно определять все файлы в каталоге.

    3. Можно выводить Warning одной процедурой.

    4. Можно работать со строками пути: извлекать каталог, извлекать расширение файла, имя и т.д.

  4. Timer.

    1. Можно создавать, удалять таймер.

    2. Можно настраивать время таймера.

    3. Можно запускать, останавливать таймер.

    4. Можно присвоить таймеру событие, чтобы оно постоянно выполнялось.

  5. GraphCanvas - мой.

    1. Присутствует прозрачность и всё для работы с ней: прозрачные цвета, все примитивы имеют функционал для рисования в прозрачном цвете.

    2. Присутствует сглаживание, так же реализовано совместно с прозрачностью. Реализовано во всех примитивах, чтобы те могли рисовать некоторые пиксели со сглаживанием и прозрачностью.

    3. Так же имеется всё остальное, что есть в GraphABC.

  6. ZenGL.

    1. В любой момент можно определить какая клавиша нажата.

    2. Больше не знаю, а что знаю, повторяет то, что выше.

  7. WinAPI.

    1. Есть события отрисовки окна.

    2. Можно создавать потоки.

    Идеи:

  1. Сделать чтобы были условия компиляции, например на define aplha, define smooth, define smoothSize 4; и ещё некоторые, которые потом можно придумать.

  2. Чтобы использовался модуль для точек point2, point3 и т.д.

  3. Так же вместе с этим модулем можно поставлять модуль для проецирования из пространств высших размерностей в более низшие.

  4. Можно сделать чтобы были классы для окружности, прямой и прочего, например в трехмерном пространстве. И чтобы их можно было сложить в массив и при помощи полиморфизма рисовать, при этом передав плоскость поворота.

    1. Для двумерных объектов можно не передавать плоскость, она будет игнорироваться.

    2. Так же может быть аргумент по умолчанию - плоскость в XY с фокусом 1 на оси z.

  5. Сделать, чтобы окно создавалось конструктором, и удалялось деструктором.

    1. При инициализации можно задавать имя окна, размеры, стиль, функции для обработки сообщений и прочее.

    2. Можно сделать структуру, чтобы ее передавать конструктору.

    3. Сделать, чтобы можно было рисовать все виды окна, возможные в WinAPI.

    4. Чтобы в структуре получались аргументы командной строки.

    Далекое будущее:

  6. Сделать, чтобы были стили рисования линии, и рассчитать их так, чтобы они хорошо рисовались на многоугольнике, или окружности, и чтобы можно было задавать размер пробелов.

  7. Сделать подробную справку по модулю, чтобы она была в формате *.chm, так же чтобы в ней было написано как компилировать программы, флаги, чтобы было написано что получилось такое с помощью WinAPI, и ещё что-то.

    Проблемы, которые надо решить:

  8. Можно ли рисовать прозрачным цветом, а потом без проблем его бтлблтить?

    1. Прозрачным цветом рисовать можно, вручную, но он не альфаблендится. Исключительно блендить своей процедурой.

    2. Если использовать прозрачность, то от стандартных примитивов можно отказаться.

      1. Или рисовать их на нижнем слое-буффере, и поверх него рисовать прозрачные цвета.

      2. Наверное это слишком затратно.(или нет, можно проверить)

  9. Надо разобраться как быть с модулем цвета и прозрачностью.

    1. Можно сделать отдельные процедуры для рисования с использованием прозрачного цвета(и рисование это будет реализовано собственными силами).

    2. И отдельные процедуры для рисования обычными виндоускими процедурами.

    3. Тогда прозрачность отделена от Непрозрачности.

    4. Прозрачный текст можно писать аналогичным методом.

    5. Было решено сделать костыль: все примитивы рисуются с нулевым цветом прозрачности. Тогда можно сделать условие, если цвет прозрачности нулевой, то сделать его 255. А истинно нулевой уровень прозрачности, когда картинка не рисуется это 0х01.

    Типы данных

  1. Буфер - содержит все связанное с графикой.

      Переменные:

    1. кисть, перо.

    1. Конструктор(размер буфера), деструктор.

    2. Забрать буфер из консольного окна.

      1. Такого не будет, для рисования в консоль надо использовать hdc консоли в битблте.

    3. Интерфейс для работы с другими буферами.

      1. Битблт

      2. Альфабленд винды.

      3. Альфабленд мой.

    4. Интерфейс для работы с кистью и пером.

      1. Настройка цвета, толщины, стиля(для альфы не будет работать).

    5. Интерфейс для вывода текста.

    6. Интерфейс для работы с рисунками.

    7. Очистка картинки.(перегрузка - задать цвет очистки)

    8. Примитивы:

      1. Поставить пиксель.

      2. Нарисовать прямоугольник

      3. Нарисовать окружность(аргументы по умочанию - цвет внутри, толщина линии, цвет пера).

      4. Нарисовать линию(аргументы по умолчанилю - цвет, толщина, стиль, типы концов).

      5. Нарисовать многоугольник(по заданному массиву вершин).

      6. Нарисовать кривую безье по координатам массива(будет очень медленная).

        1. Есть вариант как со сглаживанием, так и без.

        2. Может какие-то перегруженные варианты типо кубической, и прочей для скорости(в далеком будущем).

  2. Общие функции виндоус:

    1. Создавать потоки.

    2. Нажата ли клавиша такая-то.

    3. Сделать фулскрин.

  3. Окно - основные операции над окном. Содержит в себе объект буфера и наследуется от обычных виндоуских функций. Априори рисуется только во второй внутренний буфер.

      Переменные:

    1. HWND, HDC

      Функции:

    1. Конструктор(размер окна, заголовок, стиль), деструктор.

    2. Нарисовать из буфера.

    3. Изменить размер окна, положение, заголовок.

    4. Назначить на события функции. События: нажатие клавиши, изменение положения мышки и прочее.

  4. Картинки - класс для преобразования картинки с экрана в картинки и анимированный гиф формат.

      Скорее всего этот класс будет не реализован в ближайшее время. А будет как лишь эскиз для будущей работы. Это можно реализовать через ffmpeg - в закладках ссылка на то как именно.

      Функции:

    1. Установить слежку за таким то окном в такой то зоне, таких то размеров, в такой-то папке назначения.

    2. Сохранить картинку(если не указывается имя, то сохраняется так, как если будет делаться анимация)(после создания гифки удаляются все ненужные фреймы).

    3. Объединить сохраненные картинки в анимированную гифку с заданными задержками между фреймами и заданным именем.

    4. При уничтожении объекта удалить так же все временные картинки, которые не объединились в гифку.

  5. Цвет - будет использоваться для винапи.

      В частности особые методы задания цвета, вроде градиентов.

      Функции:

    1. Обычный rgb. Или argb.

    2. Серый из одного числа.

    3. Радуга из числа и его максимума.

    4. Циклическая радуга.

    5. Радуга, которая начинается с серого.

    6. Градиент на основе вектора цветов(расстояние между цветами одинаково).

    7. Наложение одного цвета на другой. Встраиваемая, максимально оптимизированная функция.
